## Overview

Wrapping an OpenAI client is the simplest way to get started with `@vita-tak/ai-logger`. The provider is auto-detected from the client instance — no manual configuration needed.

## Usage
```typescript
import { AILogger } from '@vita-tak/ai-logger'
import OpenAI from 'openai'

const aiLogger = new AILogger({
  apiKey: process.env.INTEGRITAS_API_KEY,
  systemId: 'your-system-name',
})

const client = aiLogger.autoLog(new OpenAI({ apiKey: process.env.OPENAI_API_KEY }))

const response = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'Hello!' }]
})

// ✓ Logged & stamped on blockchain automatically
```

<Warning>
  Always use the wrapped `client` variable for your calls — not the original `openai` instance. Calls made through the unwrapped instance will not be logged.
</Warning>

## Works with any setup

It does not matter whether your application is built with Express, Next.js, a CLI tool, or a plain script. As long as every AI call passes through the wrapped client, it will be captured and logged.
```typescript
// Express route — works fine
app.post('/chat', async (req, res) => {
  const response = await client.chat.completions.create({
    model: 'gpt-4o-mini',
    messages: [{ role: 'user', content: req.body.message }]
  })
  res.json({ reply: response.choices[0]?.message.content })
})
```

## Works with AI agents too

`autoLog()` captures every call the agent makes — regardless of which tool or step triggers it.
```typescript
const response = await client.chat.completions.create({
  model: 'gpt-4o',
  tools: [...],
  messages: [...]
})

// ✓ All agent calls logged automatically
```

<Info>
  This applies to custom-built agents and simple orchestration logic. For LangChain, see the dedicated guide.
</Info>
